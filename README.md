# Cash-Allowed Target-Vol Robust ETF Allocation

This repository implements a reproducible research workflow for:

```text
Cash-Allowed One-Year Lookback Volatility-Targeted
Robust Forecast-Aware Portfolio Optimization
for Multi-Asset ETF Allocation
```

The proposed strategy is the cash-allowed 20-day EWMA risk and dynamic-target-vol
robust forecast-aware model. The empirical comparison uses two simple baselines:
monthly rebalanced equal weight and long-only minimum variance.

No empirical result in the paper should be typed manually. Tables and figures must be
generated by the scripts in this repository.

## Asset Universe

Risky ETFs:

```text
SPY, QQQ, IWM, EFA, EEM, TLT, IEF, SHY, LQD, HYG, GLD, DBC, VNQ
```

Cash sleeve:

```text
BIL
```

`BIL` is used only as the cash-return proxy in the backtest. It is excluded from
alpha signal construction, risky covariance estimation, and the risky equal-weight
volatility stress benchmark.

The default data sample is truncated at `2026-01-31`. The Yahoo Finance downloader
uses `--end 2026-02-01` by default because the end date is exclusive.

## Proposed Strategy

Each month, after the last trading day close, the strategy uses only data observable
up to that date to estimate signals, risk, market stress, and target weights. The new
weights are applied from the next trading day and held out-of-sample until the next
monthly rebalance.

Alpha construction uses a one-year historical information window wherever applicable:
12-1 momentum uses 252-day price history skipping the most recent 21 trading days,
and trend uses a 200-day moving average. Risk control uses shorter-horizon
information: the proposed strategy estimates an annualized 20-trading-day EWMA
covariance matrix and compares 20-day EWMA risky equal-weight volatility with a
252-day risky equal-weight reference volatility.

For risky ETF `i`:

```text
momentum_i,t = P_i,t-21 / P_i,t-252 - 1
score_i,t = z(momentum_i,t) + 0.5 * trend_i,t
alpha_hat_i,t = 0.03 * score_i,t
```

The optimization allocates between risky ETF weights `x` and cash weight `c`:

```text
maximize
    alpha_hat_t' x
  - lambda_tc * 0.5 * ||[x, c] - [x_pre, c_pre]||_1
  - eta_l2 * ||x||_2^2
  - rho_t * ||Sigma_sqrt,t x||_2

subject to
    1' x + c = 1
    x >= 0, c >= 0
    x_i <= 0.25
    ||Sigma_sqrt,t x||_2 <= sigma_star_t
```

Default parameters:

```text
alpha_scale = 0.03
sigma_star_t = clip(0.10 / v_t, 0.04, 0.16)
lambda_tc = 0.001
eta_l2 = 0.01
transaction_cost_rate = 0.001
rho_t = clip(0.08 * v_t, 0.03, 0.25)
```

The cvxpy implementation uses the SOC-compatible risk expression
`cp.norm(Sigma_sqrt @ x, 2)`, not `sqrt(quad_form(...))`.

## Notes
